Welcome to ikvjwd!

The symptom

IPv6-only systems can't use GitHub.

$ git clone git@github.com:atom/atom.git
Cloning into 'atom'...
ssh: connect to host github.com port 22: Network is unreachable
fatal: The remote end hung up unexpectedly

The problem

github.com is only accessible over IPv4.

$ host -t aaaa github.com
github.com has no AAAA record

The right solution

GitHub should be available over IPv6. Only GitHub can make that happen, so contact them if IPv6 is important to you.

A workaround

Just use ikvjwd.com instead of github.com.

$ git clone git@ikvjwd.com:atom/atom.git

Security

You must verify that you see the same SSH host key fingerprint from ikvjwd.com as you are seeing from github.com. Otherwise, we could be performing a MITM attack on you. We're not doing that, but you have no reason to trust us. So please check. Seriously.

Why "ikvjwd"?

GitHub only provides IPv4. This service provides IPv6. The difference is 2.

$ echo github | caesar 2
ikvjwd

Asked Questions

How does ikvjwd work?

It uses haproxy. The config file is available on GitHub. I'm not an haproxy expert. If I could be doing it better, please open an issue and/or submit a pull request.

I get a "permission denied" error. What's going on?

GitHub requires authentication for all SSH connections, so your git client must supply a private key that is associated with a GitHub account. If you don't present a key, or if your key is not associated with a GitHub account, you will receive a fairly unhelpful error message.
